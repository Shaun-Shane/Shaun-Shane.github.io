D Moving Piece

Description

有 nnn 个格子排成一列。给定 1,2,...,n1, 2, ..., n1,2,...,n 的排列 P1,P2,...,PnP_1, P_2, ..., P_nP1​,P2​,...,Pn​，以及数组C1,C2,...,CnC_1, C_2, ..., C_nC1​,C2​,...,Cn​，表示格子 iii 能达到 PiP_iPi​，获得的分数为 CPiC_{P_i}CPi​​。从任意一点出发走 KKK 步，求能获得的最大分数。

Solution

O(N2)O(N^2)O(N2)：
以每个点作为起点 ststst，求出路径，记录环的起点 cyBegcyBegcyBeg 和终点 cyEndcyEndcyEnd (终点指再走一步能回到环的起点的节点)。
求起点 ststst 到 cyEndcyEndcyEnd 的 CCC 前缀和，以及前缀和中的最大值。并求出环上的 CCC 总和，以及从 cyBegcyBegcyBeg 开始的前缀和及其最大值。
判断能否走完整个环，若环为正环，则可以一直走到若干圈直到不能再走完整个环，或者少走一圈再加上前缀和最大值 (虽然环是正环，但可能环上存在一个极小值，少走一圈反而能使答案更大)。
O(nlogn)O(nlogn)O(nlogn)：
通过 PPP 求出每个点的入度，从入度为 0 的点开始 dfsdfsdfs 建图 (若没有点入度为 0，则所有点在环上，任意找一个点为起点即可)，并记录每个点是否在环上。分起点在环上和不在环上，起点在环上需分段讨论，然后可用支持区间修改查询的数据结构维护区间最大值。也有multiset做法 link。

F Making Palindrome

Description

求将一些字符串 S1,S2,...,SnS_1, S_2, ..., S_nS1​,S2​,...,Sn​ 拼接为回文串的最小代价 ( SiS_iSi​ 可多次被选，每选一次代价为 CiC_iCi​ )。不能拼成回文串则输出 -1。

Solution

将目标回文串分为左右两部分，设左边的字符串为 l=sl1,sl2,...,sljl = s_{l_1}, s_{l_2}, ..., s_{l_j}l=sl1​​,sl2​​,...,slj​​，右边的字符串为 r=srk,...,sr2,sr1r = s_{r_k}, ..., s_{r_2}, s_{r_1}r=srk​​,...,sr2​​,sr1​​。要使 l,rl, rl,r 构成回文串，则需要 sl1s_{l_1}sl1​​ 的前缀与 sr1s_{r_1}sr1​​ 回文，或者 sl1s_{l_1}sl1​​ 与 sr1s_{r_1}sr1​​ 的后缀回文，并且将回文部分消去后的 l,rl, rl,r 也满足上述条件 (此时 slis_{l_i}sli​​ 和 sris_{r_i}sri​​ 可能只是 SSS 的一部分)。
于是可以考虑设当前状态为 state={stringl,stringr}state = \{ string_l, string_r \}state={stringl​,stringr​}，向字符串长度小的一边加入新字符串进行拼接，并消去回文部分，直到 l,rl, rl,r 都为空，或者一个是回文串，一个是空串为止，然后跑 dijkstradijkstradijkstra 即可。
