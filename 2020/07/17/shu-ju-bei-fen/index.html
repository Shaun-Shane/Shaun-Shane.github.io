Description

有 NNN 座办公楼位于同一条街上，已知每座办公楼距离大街起点的距离。现要用 KKK 条网络电缆将办公楼连接，且任意一个办公楼都属于唯一配对组(即连接的 2K2K2K 个办公楼一定是相异的)。求 KKK 条网络电缆最小的长度总和。

Solution

容易发现，最优解中每两个配对的办公楼一定是相邻的。于是先求出两个相邻办公楼之间的距离，记为 D1,D2,D3,...,DN−1D_1, D_2, D_3, ..., D_{N - 1}D1​,D2​,D3​,...,DN−1​。于是问题可以转化为：从数列 DDD 中选取不超过 KKK 个不相邻的数，使它们的和最小。

考虑贪心：

  • 如果 K=1K=1K=1，答案显然是数列 D 中的最小值
  • 如果 K=2K=2K=2，可以证明答案是以下两种情况之一：
    • 选取 DiD_iDi​，以及除了 Di−1,Di,Di+1D_{i-1}, D_i, D_{i+1}Di−1​,Di​,Di+1​ 之外其他数中的最小值
    • 选取最小值 DiD_iDi​左右两侧的两个数，即 Di−1D_{i-1}Di−1​ 和 Di+1D_{i+1}Di+1​
    • 即最优解中，最小值左右两侧的数要么选，要么都不选
  • 为防止选取局部最优解，可以先选取数列 DDD 中的最小值 DiD_iDi​，然后把 Di−1,Di,Di+1D_{i-1}, D_i, D_{i+1}Di−1​,Di​,Di+1​ 删除，把 Di−1+Di+1−DiD_{i-1}+D_{i+1}-D_iDi−1​+Di+1​−Di​ 插入到数列 DDD 中之前执行删除的位置。这样如果之后选了 Di−1+Di+1−DiD_{i-1}+D_{i+1}-D_iDi−1​+Di+1​−Di​，相当于没有选 DiD_iDi​，换上了 Di−1D_{i-1}Di−1​ 和 Di+1D_{i+1}Di+1​。

于是可建立一个初始有 N−1N-1N−1 个节点的链表，链表节点记录数值 DiD_iDi​，节点编号 IdiId_iIdi​，以及链表节点指针或对应的迭代器。再建立一个优先队列(小根堆)，将链表节点存入。K 次操作中，取出堆顶的链表节点 ListNodeListNodeListNode，在链表中删除 ListNode−>prevListNode->prevListNode−>prev 和 ListNode−>nextListNode->nextListNode−>next，并创建新节点(数值为 ListNode−>prev−>data+ListNode−>next−>data−ListNode−>dataListNode->prev->data+ListNode->next->data-ListNode->dataListNode−>prev−>data+ListNode−>next−>data−ListNode−>data)插入链表和优先队列。

注意：为防止越界或简化边界情况判断，可以在链表起始和末尾插入极大值。
