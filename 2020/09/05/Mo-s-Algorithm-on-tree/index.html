树分块

这里介绍三种分块方法

  • 按 DFSDFSDFS 序分块。严格保证块大小为 N\sqrt NN​，但是不保证直径，也不保证联通。处理子树信息比较方便。
  • 王室联邦分块。如果确定每个块大小至少为 N\sqrt NN​，可以保证分得每个块的大小和直径都不超过 3N3\sqrt N3N​，但是不保证块联通。
  • 按深度分块。待补充…

这样分块是为了莫队的排序，使在两个查询间转移时只需进行 N\sqrt NN​ 数量级的操作。

P2325 [SCOI2005]王室联邦

Description

题目链接

Solution

  • 构造方式：

    dfsdfsdfs，同时创建一个栈存储未分块的节点。每 dfsdfsdfs 到一个点时获取当前栈的大小，当搜完当前节点 nownownow 的一个子节点后，判断栈的大小的增量是否大于等于 BBB，是则将栈内所有新增点分到新的块中，并将首都点为当前节点 nownownow，否则保留新增节点，处理下一个子节点。当当前节点的所有子节点都被搜过后，将当前节点入栈。dfsdfsdfs 结束后将栈内所有剩余节点归入已经分好的最后一个块即可。

  • 证明：

    • 显然，每个块的大小 ∣S∣≥B|S| \ge B∣S∣≥B。

    • 由上述构造方式可知 nownownow 的子节点 vvv 对应子树中剩下的未被分块的节点个数小于 BBB，而每次栈中剩下元素，即之前处理完的子树中未被分块的节点，其数目是小于 BBB 的，故合并后得到的块的大小小于 2B2B2B。

    • dfsdfsdfs 结束后，已经分好的块的大小都小于2B2B2B, 剩下的未分块的节点数小于 BBB, 故最后一个块的大小小于 3B3B3B。

    • 综上可知，每个块大小在 [B,3B][B,3B][B,3B] 之间，满足题目要求。

考虑把树上的问题转化到序列上。

这里引入欧拉序，它的思想是：当访问到点iii 时，加入序列，记录时间戳 in[i]in[i]in[i]，然后访问 iii 的子树，当访问完时，再把 iii 加入序列，并记录时间戳 out[i]out[i]out[i]。如果将一条链映射到序列中的一段区间，那么这个区间中出现两次的节点一定不属于这条链（两端点的 lcalcalca）除外, 这样只需要统计出现一次的节点即可。

对于一条链 (u,v)(u,v)(u,v)，设 in[u]<in[v]in[u]<in[v]in[u]<in[v]，讨论两种情况：
