E Stronger Takahashi

Description

题目链接

Solution

建图后跑 DijkstraDijkstraDijkstra 或者 01BFS01BFS01BFS，具体建法如下：

  • 设当前位置为 (i,j)(i,j)(i,j)。
  • 由 (i,j)(i,j)(i,j) 向四周的 '.' 连代价为 0 的边。这些点不需要 punch 即可到达。
  • 由 (i,j)(i,j)(i,j) 向四周的 '#' 的 3×33×33×3 范围内的所有点连代价为 1 的边。若经由 '#' 到这些点，需要 punch 一次。

F Common Prefixes

Description

题目链接

Solution

本题求两个后缀的 LCPLCPLCP，等价于将原串翻转后，求某两个前缀的 LCSLCSLCS。我们知道 SAMSAMSAM 的后缀树中，原串两个前缀对应节点的 LCALCALCA 是这两个前缀的 LCSLCSLCS 对应的节点（该 LCSLCSLCS 一定是这个状态中最长的一个串）。利用这个性质可以求解本题。

令 G(Sk)=f(Sk,S1)+f(Sk,S2)+…+f(Sk,SN)G(S_k)=f(S_k,S_1)+f(S_k,S_2)+\ldots+f(S_k,S_N)G(Sk​)=f(Sk​,S1​)+f(Sk​,S2​)+…+f(Sk​,SN​)。显然对每个 SkS_kSk​ 求 f(Sk,S1),f(Sk,S2),…,f(Sk,SN)f(S_k,S_1),f(S_k,S_2),\ldots,f(S_k,S_N)f(Sk​,S1​),f(Sk​,S2​),…,f(Sk​,SN​) 效率太低。我们不妨考虑每个 LCSLCSLCS 对 G(S1),…,G(Sn)G(S_1),\dots,G(S_n)G(S1​),…,G(Sn​) 的贡献。具体的，我们预处理出后缀树上每个节点 uuu 的子树中包含前缀的数目，记为 fuf_ufu​，那么 uuu 对儿子 vvv 的子树中所有前缀的答案贡献为 (fu−fv)∗maxsubu(f_u-f_v)*maxsub_u(fu​−fv​)∗maxsubu​。此外，若 uuu 包含了某一前缀 SiS_iSi​，那么 G(Si)G(S_i)G(Si​) 还要加上 fu∗maxsubuf_u*maxsub_ufu​∗maxsubu​。

上述解法涉及子树修改加单次查询，利用 DFSDFSDFS 序 + 差分即可。

Solution

设 f[S]f[S]f[S] 表示点集为 SSS 的联通子图的个数，g[S]g[S]g[S] 表示点集为 SSS 的子图的个数，那么答案可以表示为 ∑{1,k}⊆S⊆Vf[S]g[V−S]\sum\limits_{\{1,k\}⊆S⊆V}f[S]g[V-S]{1,k}⊆S⊆V∑​f[S]g[V−S]。下面考虑如何求解 f[S],g[S]f[S],g[S]f[S],g[S]。

设有 mmm 条边两端都在 SSS 中，则 g[S]=2mg[S]=2^mg[S]=2m。求 f[S]f[S]f[S] 可以利用容斥，从 g[S]g[S]g[S] 中扣除不联通的部分，而不联通的部分可以拆成一个联通子图和一个另一个子图，因此可以粗略得到 f[S]=g[S]−∑T⊆Sf[T]g[S−T]f[S] = g[S] - \sum\limits_{T⊆S} f[T]g[S-T]f[S]=g[S]−T⊆S∑​f[T]g[S−T]，这可通过枚举子集求得。由于 g[S]g[S]g[S] 是包含 f[S]f[S]f[S] 的，因此 f[T]g[S−T]f[T]g[S-T]f[T]g[S−T] 和 f[S−T]g[T]f[S-T]g[T]f[S−T]g[T] 都包含了 f[T]f[S−T]f[T]f[S-T]f[T]f[S−T]，这一部分要扣除，故当 T>S−TT>S-TT>S−T 时，要从 g[S−T]g[S-T]g[S−T] 中扣除 f[S−T]f[S-T]f[S−T]。

求 g[S]g[S]g[S] 的时间复杂度为 O(m2n)O(m2^n)O(m2n)，求 f[S]f[S]f[S] 的时间复杂度为 O(3n)O(3^n)O(3n)，因此总的时间复杂度为 O(3n+m2n)O(3^n+m2^n)O(3n+m2n)。
